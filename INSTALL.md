# Краткая инструкция по установке

Этот проект состоит из стандартного WordPress, базы MySQL/MariaDB, кастомной темы и кастомного плагина. Отдельный backend, Node.js, Python, Redis или сервис двухфакторной аутентификации для работы сайта не требуются.

## Что передать на сервер

- `outputs/fyremezzonine-wp-theme.zip` - тема сайта.
- `outputs/fyremezzonine-conference-manager.zip` - плагин конференций, заявок, партнеров, ролей и email-подтверждения.
- Доступ к MySQL/MariaDB для WordPress.
- SMTP-доступ к почте отправителя, например `noreply.oren@vniipo.ru`.

## Установка на чистый WordPress

1. Установить WordPress на сервере и подключить его к MySQL/MariaDB.
2. Войти в админку WordPress под администратором.
3. Открыть `Внешний вид -> Темы -> Добавить новую -> Загрузить тему`.
4. Загрузить `outputs/fyremezzonine-wp-theme.zip` и активировать тему.
5. Открыть `Плагины -> Добавить новый -> Загрузить плагин`.
6. Загрузить `outputs/fyremezzonine-conference-manager.zip` и активировать плагин.
7. Открыть `Настройки -> Постоянные ссылки` и нажать `Сохранить изменения`, чтобы WordPress обновил маршруты.
8. Создать или проверить страницы:
   - `/registration/` со shortcode `[conference_registration_form]`;
   - `/partnership/` со shortcode `[conference_partner_request_form]`;
   - `/editor/conferences/` со shortcode `[conference_submission_form]`.
9. Создать пользователей-редакторов и выдать им роль с правом редактировать конференции.
10. В разделе `Редактор -> Конференции` создать конференцию, заполнить данные, проверить предпросмотр и опубликовать.

## Настройка почты

Плагин отправляет участникам шестизначный код подтверждения после регистрации. До подтверждения email заявка хранится как `pending_email` и не попадает в рабочие таблицы, печать, Excel и статистику секций.

Для отправки писем WordPress должен видеть SMTP-переменные окружения:

```text
FYREMEZZONINE_SMTP_HOST=smtp.mail.ru
FYREMEZZONINE_SMTP_PORT=2525
FYREMEZZONINE_SMTP_ENCRYPTION=tls
FYREMEZZONINE_SMTP_USERNAME=noreply.oren@vniipo.ru
FYREMEZZONINE_SMTP_PASSWORD=<пароль приложения SMTP>
FYREMEZZONINE_SMTP_FROM_EMAIL=noreply.oren@vniipo.ru
FYREMEZZONINE_SMTP_FROM_NAME=ВНИИПО Конференции
```

Пароль не хранить в Git, README, архивах темы или плагина. На локальном Docker-стенде значения берутся из `work/.env`, шаблон лежит в `work/.env.example`.

Если сервер не Docker, переменные нужно задать в панели хостинга, конфигурации PHP-FPM/Apache/Nginx или в `wp-config.php` через `putenv(...)` до строки `/* That's all, stop editing! */`.

## Проверка после установки

1. Открыть главную страницу и убедиться, что показывается текущая опубликованная конференция.
2. Отправить тестовую заявку на участие на реальный email.
3. Проверить, что письмо с кодом пришло.
4. Ввести код на `/registration/verify/`.
5. Убедиться, что после подтверждения показывается сообщение об успешной регистрации и ссылка на чат MAX.
6. В админке открыть `Редактор -> Заявки на участие` и проверить, что подтвержденная заявка появилась в таблице.
7. Проверить Excel-выгрузку, печать, фильтр по конференции и фильтр по тематике/секции.

## Обновление уже установленного сайта

1. Сделать резервную копию базы данных и папки `wp-content`.
2. Загрузить новую версию ZIP-плагина через `Плагины -> Добавить новый -> Загрузить плагин`.
3. При запросе WordPress заменить старую версию новой.
4. Загрузить новую версию ZIP-темы через `Внешний вид -> Темы -> Добавить новую -> Загрузить тему`.
5. Проверить главную страницу, регистрацию, заявки, партнерство и редакторский раздел конференций.

## Минимальные требования

- PHP версии, совместимой с текущим WordPress.
- WordPress.
- MySQL или MariaDB.
- Рабочая отправка исходящей почты через SMTP.
- Включенный WP-Cron или системный cron для очистки неподтвержденных заявок старше суток.
