# WordPress Conference Service

CMS-сервис для управления сайтами конференций на WordPress. Проект создан как замена статичной/самописной странице: редакторы могут создавать новые конференции, менять ключевой контент, управлять регистрацией и смотреть заявки через админку WordPress без правки кода.

Краткая инструкция по установке и обновлению вынесена в отдельный файл `INSTALL.md`.

## Что входит в проект

- `outputs/fyremezzonine-wp-theme/` - кастомная WordPress-тема конференционного сайта.
- `outputs/fyremezzonine-conference-manager/` - кастомный плагин для конференций, полей, регистрационной формы и заявок.
- `work/docker-compose.yml` - локальная Docker-среда WordPress + MariaDB.
- `outputs/fyremezzonine-wp-theme.zip` - ZIP темы для установки в WordPress.
- `outputs/fyremezzonine-conference-manager.zip` - ZIP плагина для установки в WordPress.
- `INSTALL.md` - короткая инструкция по установке на сервер, обновлению и настройке SMTP.
- `outputs/cms-wordpress-service-defense.pptx` - подробная презентация по архитектуре, CMS-подходу, user flow, БД, ролям и roadmap.

## Ключевая идея

Контентная часть сайта автономна и управляется через CMS:

- редактор создает новую конференцию в WordPress;
- заполняет название, описание, даты, дедлайн и одно или несколько мест проведения;
- меняет изображения, темы обсуждения, материалы, программу и карту;
- публикует конференцию без участия разработчика;
- редактор явно завершает текущую конференцию, после чего следующая опубликованная конференция появляется на главной;
- регистрация открыта только на текущую конференцию и дополнительно ограничена дедлайном;
- заявки участников сохраняются в базе данных.

IT-специалисты остаются нужны для production-инфраструктуры, безопасности, обновлений, резервного копирования, изменения бизнес-логики и крупных доработок дизайна.

## Локальный запуск

Требования:

- Docker Desktop;
- Git;
- свободный порт `8080`.

Запуск:

```powershell
cd work
Copy-Item .env.example .env # только при первой настройке; заполните SMTP-секреты
docker compose up -d
```

Локальные SMTP-настройки находятся в `work/.env`. Этот файл исключен из Git. В репозитории хранится только безопасный шаблон `work/.env.example` без пароля. Для корпоративной почты проекта используется SMTP Mail.ru: `smtp.mail.ru`, резервный порт `2525` и STARTTLS, поскольку стандартные SMTP-порты в локальной сети недоступны. Пароль рекомендуется выпускать как отдельный пароль для внешнего приложения.

Система почтового подтверждения регистрации уже реализована внутри плагина. После отправки заявки участник получает шестизначный код на email, подтверждает заявку на `/registration/verify/`, и только после этого заявка попадает в рабочие списки, статистику, печать и Excel. Неподтвержденные заявки старше суток очищаются через WP-Cron.

Адреса:

- сайт: `http://127.0.0.1:8080/`
- админка: `http://127.0.0.1:8080/wp-admin/`
- страница регистрации: `http://127.0.0.1:8080/registration/`
- архив конференций: `http://127.0.0.1:8080/conferences/`

Локальные учетные записи создаются в базе WordPress. Пароли намеренно не хранятся в репозитории и должны передаваться пользователям отдельно.

## Архитектура

```text
Посетитель
  -> WordPress
    -> Theme: внешний вид, главная, архив, страница конференции
    -> Conference Manager: тип "Конференции", поля, регистрация, заявки
    -> MariaDB: записи, метаполя, пользователи, заявки
```

Основные таблицы:

- `wp_posts` - страницы WordPress и записи типа `conference`;
- `wp_postmeta` - поля конференций: даты, дедлайн, город, место, карта, изображения, ссылки;
- `wp_users` и `wp_usermeta` - администраторы и редакторы;
- `wp_options` - настройки WordPress и темы;
- `wp_options.fyremezzonine_partner_catalog` - общий каталог партнеров с типом, сайтом, логотипом и контактами;
- `wp_conference_registrations` - заявки участников, включая множественный выбор интересующих тем и секций;
- `wp_conference_partner_requests` - заявки компаний на партнерство.
- `wp_conference_activity_log` - журнал создания, публикации, изменений, снятия с публикации и завершения конференций.

## Модульность плагинов

Базовая предметная область находится в доменном плагине `fyremezzonine-conference-manager`. Он хранит связанную бизнес-логику конференций, регистрации и редакторского интерфейса, а внешний вид остаётся в теме.

Сейчас плагин отвечает за связанный набор задач:

- регистрацию типа записей `conference`;
- метаполя конференции;
- форму регистрации участников;
- сохранение заявок;
- список заявок в админке;
- управляемый жизненный цикл конференции и журнал действий;
- проверку текущей конференции и дедлайна регистрации;
- инструкцию для редакторов.

То есть это не случайный набор несвязанных функций, а один предметный модуль `conference management`.

При росте проекта плагин можно декомпозировать на внутренние файлы:

```text
fyremezzonine-conference-manager/
  fyremezzonine-conference-manager.php
  includes/
    post-type.php
    meta-fields.php
    registration-form.php
    registrations-admin.php
    editor-guide.php
    database.php
```

Если система станет крупнее, возможна и более жесткая декомпозиция на отдельные плагины:

```text
conference-manager
  - тип конференций
  - поля конференций
  - очередь публикаций и журнал действий

conference-registration
  - форма регистрации
  - таблица заявок
  - статусы заявок
  - экспорт заявок

conference-editor-guide
  - инструкции для редакторов
  - подсказки в админке
```

Текущая формулировка архитектурного решения:

> Управление конференциями остаётся одним доменным модулем, а представление сайта находится в теме. Граница между бизнес-логикой и отображением сохраняется через стандартные API WordPress.

Следующим техническим шагом при росте проекта будет разнести крупный файл менеджера конференций по внутренним `includes/`, не меняя публичные функции между модулем и темой.

## Что может делать редактор

Через раздел `Конференции` в админке редактор может:

- создать новую конференцию;
- заполнить даты и дедлайн регистрации;
- добавить несколько мест проведения с названием, городом, целью площадки, точным адресом, отдельной картой и маршрутом;
- задать ссылку на программу;
- поменять фон первого экрана: статичное изображение или GIF-заставку;
- добавлять любое количество тем обсуждения через `+ Добавить тему`, указывать изображение и создавать отдельные секции внутри каждой темы через `+ Добавить секцию`;
- обновить текст блока "О конференции";
- добавить тезисы/преимущества;
- добавлять спикеров конференции с фото, именем, должностью и выдержкой из их слов;
- добавлять организаторов через `+ Добавить организатора`, а генеральных партнеров, партнеров и информационных партнеров через `+ Добавить партнера`;
- настроить отдельную Яндекс.Карту по адресу каждой площадки;
- загрузить два изображения, которые показываются под картой;
- опубликовать конференцию и проверить отображение на сайте;
- снять текущую или будущую конференцию с публикации;
- завершить текущую конференцию и передать главную страницу следующей;
- удалить ненужную конференцию в корзину WordPress;
- посмотреть дату начала подготовки, первой публикации, последнего изменения и журнал действий.

В форме регистрации участник отмечает несколько интересующих тем или секций. Если у темы секции не заполнены, сама тема автоматически используется как единственный вариант выбора. Выбранные значения сохраняются вместе с заявкой и попадают в таблицу, печать и Excel.

Все роли WordPress, которым разрешено редактировать записи, автоматически получают отдельные права на управление конференциями. Основная рабочая область редактора находится на странице `/editor/conferences/`: там объединены создание, выбор и изменение конференций. Стандартный список WordPress остается доступен администраторам через админку, но для ежедневной работы редактору он не нужен.

## Единый раздел конференций

После входа редактор открывает в шапке:

```text
Редактор -> Конференции
```

На странице конференции разделены на проходящие сейчас, будущие, черновики и завершенные. Кнопка `Создать новую` открывает пустую анкету. Нажатие на текущую, будущую конференцию или черновик открывает ту же анкету с уже заполненными данными. Завершенные конференции показаны в списке без возможности редактирования.

Редактор заполняет поля, сохраняет черновик, проверяет предпросмотр и публикует запись типа `conference`. Первая опубликованная конференция становится текущей, остальные попадают в очередь будущих. После нажатия `Завершить конференцию` следующая опубликованная запись становится текущей независимо от дат начала и окончания.

Прямой адрес:

```text
/editor/conferences/
```

Если нужна отдельная страница на сайте, например `Новая конференция`, вставьте в нее shortcode:

```text
[conference_submission_form]
```

Форма доступна только вошедшим пользователям с правами редактирования записей. Для локальной демонстрации можно использовать пользователя:

```text
conference_editor / editor12345
```

## Регистрация участников

Плагин добавляет shortcode:

```text
[conference_registration_form]
```

Форма сохраняет:

- конференцию;
- фамилию, имя и отчество отдельными полями;
- должность;
- email;
- телефон;
- организацию;
- обязательное согласие на обработку персональных данных и с политикой сайта;
- обязательное ознакомление и согласие с официальной политикой конфиденциальности MAX, поскольку чат конференции размещается в MAX;
- статус заявки;
- IP-адрес;
- дату создания.

Без любой из двух отметок сервер отклоняет регистрацию, даже если обязательность полей была обойдена в браузере. Факты согласия сохраняются в MySQL для аудита, но не выводятся в рабочей таблице заявок и Excel-выгрузке.

После отправки формы участник получает на указанный email шестизначный код. До ввода кода заявка хранится со статусом `pending_email`, не попадает в рабочую таблицу, статистику, печать и Excel. Код действует 10 минут, допускается не более пяти попыток ввода и трех отправок за один период. После подтверждения заявка получает рабочий статус, а участнику показывается ссылка на чат MAX. Неподтвержденные заявки старше суток автоматически удаляются через WP-Cron.

Плагин автоматически создаёт редактируемую страницу `Политика обработки персональных данных`, назначает её штатной страницей конфиденциальности WordPress и выводит ссылку в регистрационной форме и подвале. Официальная политика MAX открывается по адресу `https://legal.max.ru/pp`.

Если дедлайн конференции уже прошел, форма не отображается, а посетитель видит сообщение `Регистрация закрыта`. Серверная проверка дедлайна также выполняется перед записью заявки в БД.

На странице отдельной конференции регистрационная форма не вставляется автоматически: сначала посетитель видит описание и данные конференции, затем сам нажимает кнопку `Зарегистрироваться`.

Блок `Требования к оформлению материалов` общий для сайта. Его текст и ссылка не заполняются заново в каждой конференции.

Заявки смотрятся на сайте через меню:

```text
Редактор -> Заявки на участие
```

Прямой адрес:

```text
/editor/registrations/
```

На странице работают два автоматических фильтра: по конференции и по выбранной теме/секции. Система показывает точное количество потенциальных участников для текущего фильтра. В каждой строке ответственный может поставить галочку `Прибыл`; без галочки участник считается не прибывшим. Отметка сразу сохраняется в БД и включается в печать и Excel. Печать и `Выгрузить Excel` используют тот же отфильтрованный список. Старые регистрации не удаляются: они остаются в таблице `wp_conference_registrations` и доступны как архив, включая прежние названия тематик.

## Ответственный за секции

Плагин регистрирует отдельную роль WordPress `Ответственный за секции` (`conference_section_manager`). Администратор может назначить ее пользователю в стандартном разделе `Пользователи`.

После входа ответственный автоматически попадает в административный раздел `Статистика секций`. В нем доступны только:

- выбор конференции;
- выбор тематики или секции;
- количество заявок, в которых участники выбрали этот вариант.

Роль не видит ФИО, email, телефоны и другие персональные данные, не может печатать или экспортировать заявки, отмечать прибытие, редактировать конференции, партнеров, пользователей, плагины и настройки WordPress. Попытка открыть другой раздел админки перенаправляет обратно в статистику секций.

## Чат участников

Редактор указывает ссылку на чат в карточке конференции. Ссылка не выводится на публичной странице конференции и появляется только в сообщении об успешной регистрации на текущую конференцию.

Интеграции с MAX, webhook и модерации доступа в проекте нет. За актуальность и настройки ссылки отвечает редактор конференции.

## Заявки на партнерство

Кнопка `Оставить заявку` на главной странице ведет на внутреннюю страницу:

```text
/partnership/
```

Это не внешняя Google форма. Форма создается плагином и сохраняет данные в таблицу `wp_conference_partner_requests`.

Форма собирает:

- название компании;
- сайт компании;
- город;
- степень партнерства;
- контактное лицо;
- должность контактного лица;
- email;
- телефон;

Редакторы управляют партнерами и заявками через меню:

```text
Редактор -> Партнерство
```

Прямой адрес:

```text
/editor/partnership/
```

При первом открытии раздела общий каталог автоматически импортируется из последней конференции, в которой заполнены организаторы и партнеры. В каталоге можно:

- открыть партнера нажатием на карточку;
- изменить название, степень партнерства, сайт, город, контакты, комментарий и логотип;
- удалить партнера из общего списка;
- добавить партнера вручную;
- одобрить входящую заявку и перенести данные компании в каталог.

Новая конференция автоматически получает партнеров из общего каталога, распределяя их по блокам `Организаторы`, `Генеральные партнеры`, `Партнеры` и `Информационные партнеры`. Изменение каталога не переписывает уже созданные конференции.

## Production-рекомендации

Перед реальным запуском нужно:

- заменить локальные пароли;
- настроить HTTPS и домен;
- настроить SMTP-переменные окружения для `noreply.oren@vniipo.ru` и использовать отдельный пароль почтового приложения;
- проверить доставляемость писем с кодами подтверждения на реальных адресах;
- добавить антиспам для формы;
- ограничить права редакторов;
- настроить резервные копии БД и `wp-content`;
- проверить мобильную версию и скорость;
- протестировать обновления WordPress/плагинов на staging-копии.

## Roadmap

Ближайшие полезные улучшения:

- заменить ручные URL изображений на выбор файлов из медиабиблиотеки;
- расширить экспорт заявок полноценным XLSX при необходимости;
- добавить отдельные email-уведомления администраторам о новых подтвержденных заявках;
- расширить инструкцию внутри админки примерами заполнения.
